# Launch Checklist

All content is drafted and ready in this directory. Work through these in order.

## Pre-Launch (Repo)

- [ ] Merge `develop` to `main` and push
- [ ] Enable GitHub Discussions on ejwinter/TeamFabric (Settings → Features → Discussions)
- [ ] Create a pinned Showcase discussion: "Share your TeamFabric instance"
  - Use GitHub Discussions → New Discussion → Show and Tell
  - Body: "Using TeamFabric with your team? Share it here — team type, modules used, anything you've customized."
  - Pin it

## Phase 1 — Claude Code Community (Launch Week)

- [ ] Post to Anthropic Discord #show-and-tell → use `discord-post.md`
- [ ] Post to r/ClaudeAI → use `reddit-post.md`
- [ ] Post to r/ChatGPTCoding → use `reddit-post.md`

## Phase 2 — Broader Developer Community (Weeks 2–4)

- [ ] Submit Show HN to Hacker News → use `hn-post.md`
  - Best time: weekday morning US time
- [ ] Submit to TLDR Newsletter → https://tldr.tech/submit → use `newsletter-submissions.md`
- [ ] Submit to Pointer Newsletter → https://www.pointer.io/submissions/ → use `newsletter-submissions.md`
- [ ] Submit to Software Lead Weekly → http://softwareleadweekly.com/submit → use `newsletter-submissions.md`

## Phase 3 — Thought Leadership (Ongoing)

- [ ] Publish LinkedIn Article 1 → use `linkedin-article-1.md`
  - Title: "Why team documentation always falls behind — and what I think changes with AI agents"
- [ ] Publish LinkedIn Article 2 → use `linkedin-article-2.md`
  - Title: "What working memory means for software teams — and why it matters more as teams grow"

## Ongoing Stewardship

- [ ] Respond to GitHub Issues and Discussions within a few days (critical in first 60 days)
- [ ] Monthly: post a GitHub Release changelog + brief note in Discord/Reddit
- [ ] As teams adopt: encourage them to add a row to SHOWCASE.md via PR
