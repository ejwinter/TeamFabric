Product: clinical-lake
Version: v3.0
Status: In Progress
Target Date: 2026-06-30
Owner: marcus.chen@riverdale.org

## Summary

v3.0 delivers two major capabilities to the Clinical Data Lake: FHIR R4 resource parsing alongside the existing HL7v2 pipeline, and a streaming-first ingestion architecture via Kafka for near-real-time clinical data availability. This release targets Clinical Informatics teams who need FHIR-format data for quality measure calculations and operational consumers who require lower-latency ADT event processing.

## Decisions

### 2026-05-10 FHIR resource scope for v3.0
- **Decided by:** Marcus Chen
- **Recorded by:** Claude (ingestion)
- **Options considered:** Full FHIR R4 resource set (rejected: too broad for a first FHIR release), Patient + Encounter + Observation only (chosen)
- **Rationale:** Scoping to three core resource types allows Clinical Informatics to validate the FHIR pipeline before broader rollout. Remaining resource types deferred to v3.1.
