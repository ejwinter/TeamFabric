# Default Workflow — Evaluation Process

## Overview
Requests following the default workflow are evaluated in two stages: an initial screening (L1) followed by a deeper team consultation (L2). Each stage has its own rubric and possible outcomes.

## Stages

### 1. L1 Screening
**Purpose:** Quick feasibility and completeness check at intake. Determines whether the request is worth committing team time to a full consultation.
**Typically performed by:** Team lead (Dana Torres)
**Rubric:** l1-rubric.md

**Outcomes and transitions:**
- **Advance to L2** → proceed to L2 Consultation
- **Needs More Information** → request goes back to requestor with specific questions; re-enter L1 when answers arrive
- **Reject** → terminal; document reason on the request entity
- **Defer** → set a follow-up date; re-enter L1 when conditions change

### 2. L2 Consultation
**Purpose:** In-depth technical evaluation after meeting with the requestor. Determines acceptance, engagement model, and staffing approach.
**Typically performed by:** Team lead + one or more team members relevant to the request
**Rubric:** l2-rubric.md

**Outcomes and transitions:**
- **Accept** → terminal; proceed to staffing and planning
- **Accept with Conditions** → terminal; document conditions, proceed to staffing once conditions are met
- **Needs More Information** → schedule follow-up with requestor; re-enter L2 when answers arrive
- **Reject** → terminal; document reason clearly and kindly
- **Defer** → set a re-evaluation date; re-enter L2 when capacity or conditions change

## Workflow-Specific Guidance

- L1 should be quick — 10-15 minutes reviewing the request. If it takes longer, the request probably needs clarification from the requestor.
- L2 should happen after a real conversation with the requestor, not based solely on the written request. If no consultation has occurred, note this before proceeding.
- When evaluating capacity impact at L2, load team.md for current allocation and engagement counts.
- The engagement model decision (Full-Service / Collaborative / Advisory) at L2 should flow naturally from the criteria. If the requestor has a data scientist, lean toward Collaborative unless complexity warrants Full-Service.
